üéØ Event-Driven Programming

üìò What is Event-Driven Programming?

A programming paradigm where the flow of execution is determined by events:

  • Events: User actions (clicks, key presses) or system events
  • Event Listeners: Objects that respond to events
  • Event Handlers: Methods called when events occur

The program waits for events rather than executing sequentially.

üìò JavaFX Application Lifecycle

When a JavaFX application starts, the following methods are called in order:

  1. init() - Optional initialization before UI is created
  2. start(Stage) - Required! Create and show the UI
  3. stop() - Optional cleanup when application closes

üìò Stage and Scene

Stage = The window (like JFrame in Swing)

JavaFX allows CSS-like styling using the setStyle() method or external CSS files. The syntax uses -fx- prefixed property names:

// Inline CSS styling for a button

    "-fx-text-fill: " + PALETTE_CREAM + "; " +
    "-fx-font-weight: bold; " +
    "-fx-font-size: 14px; " +
    "-fx-cursor: hand;";
button.setStyle(baseStyle);
