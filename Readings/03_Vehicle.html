What is this class?
Vehicle is a subclass of BookableItem that represents a vehicle for rent. It demonstrates advanced polymorphism by overriding the canBeBookedBy() method to add custom validation.

üéØ Key OOP Concepts

üìò Definition: Polymorphism

Polymorphism (Greek: "many forms") allows objects of different classes to be treated through the same interface. There are two types:

  • Compile-time (Method Overloading): Same method name, different parameters
  • Runtime (Method Overriding): Subclass provides different implementation of parent method

Vehicle demonstrates runtime polymorphism - same method call (canBeBookedBy()) behaves differently for Vehicle vs. other BookableItems.

üìò Calling Parent Method with super.methodName()

When overriding a method, you can still call the parent's implementation using super.methodName(). This allows:

  • Extension: Add new behavior while keeping the original
  • Reuse: Don't repeat code from parent

How this method works:

  1. super.canBeBookedBy(user) calls parent's version (checks availability and user != null)
  2. If basic check fails, return false immediately
  3. Additional Vehicle-specific check: user must have 5+ loyalty points
  4. If points check fails, print message and return false
  5. All checks passed: return true

    if (item.canBeBookedBy(user)) {  // Called polymorphically
        item.bookItem(user);
    }
