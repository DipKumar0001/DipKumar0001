What is this class?
ItemFactory implements the Factory Method Design Pattern - a creational pattern that encapsulates object creation logic.

ğŸ¯ Design Pattern: Factory Method

ğŸ“˜ Factory Method Pattern

Category: Creational Design Pattern

Intent: Define an interface for creating an object, but let subclasses or factory methods decide which class to instantiate.

Benefits:

  • Encapsulates object creation in one place
  • Client code doesn't need to know concrete classes
  • Easy to add new product types
  • Promotes loose coupling

ğŸ“Š Usage Comparison

Without Factory (Direct Instantiation)

// Client code KNOWS and DEPENDS on concrete classes
Book book = new Book("B001", "Alchemist", "Paulo", "978...");
Vehicle car = new Vehicle("V001", "Honda", "Honda", "Civic", "AB12");

With Factory (Encapsulated Creation)

// Client code uses factory - creation logic is centralized
ItemFactory factory = new ItemFactory();
Book book = factory.createBook("B001", "Alchemist", "Paulo", "978...");
Vehicle car = factory.createVehicle("V001", "Honda", "Honda", "Civic", "AB12");
