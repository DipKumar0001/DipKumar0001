ğŸ“˜ Definition: Inheritance

Inheritance is a mechanism where a new class (child/subclass) is derived from an existing class (parent/superclass). The child class:

  • Inherits all public and protected fields and methods from the parent
  • Can add new fields and methods
  • Can override (replace) inherited methods

IS-A Relationship: A Book IS-A BookableItem (Book is a type of BookableItem)

ğŸ“˜ Definition: Constructor Chaining (super)

The super() keyword calls the parent class's constructor. Rules:

  • super() MUST be the first statement in the constructor
  • If not explicitly called, Java automatically inserts super() (no-arg version)
  • If parent has no no-arg constructor, you MUST call super() with appropriate arguments

What happens when new Book("B001", "Alchemist", "Paulo Coelho", "978...") is called:

  1. Book constructor is invoked
  2. super(itemId, name) calls BookableItem constructor
  3. BookableItem constructor sets itemId, name, isAvailable, currentHolder
  4. Control returns to Book constructor
  5. author and isbn are set

ğŸ“˜ Definition: Method Overriding

Method overriding occurs when a subclass provides its own implementation of a method that is already defined in its parent class. Rules:

  • Same method signature (name + parameters)
  • Same or covariant return type
  • Access modifier cannot be more restrictive
  • Use @Override annotation (optional but recommended)

This is POLYMORPHISM! Different classes respond differently to the same method call.

Key observations:

  • @Override tells compiler to check the parent has this method
  • getName(), getItemId(), isAvailable() are inherited from parent
  • The implementation is Book-specific, printing author and ISBN
