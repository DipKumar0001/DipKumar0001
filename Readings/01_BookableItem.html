What is this class?
BookableItem is an abstract base class that represents any item that can be booked in the library system. It defines common properties and behaviors shared by all bookable items (Books, Vehicles, HobbyKits).

üéØ Key OOP Concepts in This Class

üìò Definition: Abstract Class

An abstract class is a class that cannot be instantiated directly. It serves as a blueprint for other classes (subclasses) to inherit from. Abstract classes can contain:

  • Abstract methods - methods without implementation that subclasses MUST override
  • Concrete methods - regular methods with implementation that subclasses inherit
  • Instance variables - fields that subclasses inherit

üìò Definition: Encapsulation

Encapsulation is bundling data (attributes) and methods that operate on that data within a single unit (class), while restricting direct access to some components. This is achieved by:

  • Making attributes private (hidden from outside)
  • Providing public getter/setter methods to access/modify them

Why? Protects data integrity, allows validation, and hides implementation details.

üìò Definition: Abstract Method

An abstract method is declared with the abstract keyword and has no body (no implementation). It ends with a semicolon instead of curly braces.

Contract: Any concrete (non-abstract) subclass MUST provide an implementation for all abstract methods.

üìò Definition: Getters and Setters

Getters (accessors) and Setters (mutators) are methods that provide controlled access to private fields:

  • Getter: Returns the value of a private field. Convention: getFieldName() or isFieldName() for booleans
  • Setter: Sets/modifies the value of a private field. Convention: setFieldName(value)

Benefits: Allows validation before setting, read-only access (no setter), lazy loading, logging changes, etc.
